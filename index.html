Subjects are the things under test — any model or agent at a remote endpoint (no LLM/agent type). A subject is its name + properties; PBIntra calls it through connector containers using your stored secrets.

Everything is a container. Components are the container images PBIntra orchestrates — a storage serves test data, a connector talks to subjects, a runner wires them together, a scorer grades each response, and an optional uploader exports results out.

component = prebuilt image hosted elsewhere · only the image URL is needed

A pipeline is the named, reusable graph of components you assemble on the canvas. Launch one to create a run.

PBIntra can run each container on one of several platforms. Use Add platform (top-right) to connect the ones you use, then pick a platform per container in the pipeline builder. Credentials here are just the access config PBIntra needs to create the job and pull the image.

Use Add container source (top-right) to connect the registries PBIntra pulls component images from. When you register a component whose image lives in a private registry, attach one of these so PBIntra has the pull credentials. Public images need none.

Triggers launch a pipeline automatically. Use Add trigger (top-right) to create one — pick a type, choose the pipeline it runs, and set its arguments. A time-based trigger fires on a schedule (an interval, a one-off date, or a recurring cron); a webhook trigger fires whenever its unique URL receives a request. Disable a trigger to pause it, or remove it entirely.
